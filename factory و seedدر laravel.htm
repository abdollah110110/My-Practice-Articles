استفاده از factory و seedدر laravel

برای استفاده از factory و seed مراحل زیر را انجام میدهیم:

1- مدل و مایگریشن را ایجاد میکنیم:

php artisan make:model Post -m

2- در فایل مایگریشن مربوطه، فیلدهای جدول را تعیین میکنیم:

public function up()
{
    Schema::create('posts', function (Blueprint $table) {
        $table->increments('id');

        $table->integer('user_id')->unsigned();
        $table->foreign('user_id')->references('id')->on('users')->onDelete('cascade');

        $table->string('title');

        $table->text('body');

        $table->timestamps();
    });
}

3- در مسیر app\database\factories و در فایل ModelFactory.php، کدهای زیر را وارد میکنیم:

$factory->define(App\Post::class, function (Faker\Generator $faker) {
    return [
        'user_id' => $faker->numberBetween($min = 1),
        'title' => $faker->sentence(),
        'body' => $faker->text(rand(250, 500)),
    ];
});

4- در مسیر app\database\seeds و در فایل DatabaseSeeder.php، کدهای زیر را وارد میکنیم:

public function run()
{
    // $this->call(UsersTableSeeder::class);
    factory(App\User::class, 5)->create()->each(function($u){
        $u->posts()->saveMany(factory(App\Post::class, 3)->make());
    });
}

5- در مسیر app\Providers و در فایل AppServiceProvider.php، کد زیر را وارد میکنیم:

public function boot()
{
    Schema::defaultStringLength(191);
}

6- در مدل User ریلیشن posts را تعیین میکنیم:

public function posts() {
    return $this->hasMany(Post::class);
}

7- در مدل Post ریلیشن user و فیلیبلها را تعیین میکنیم:

    protected $fillable = ['user_id', 'title', 'body'];
    
    public function user() {
        return $this->belongsTo(User::class);
    }

8- در خط فرمان دستورات زیر را به ترتیب اجرا میکنیم:

php artisan migrate

php artisan db:seed

حالا اگر به phpmyadmin برویم خواهیم دید که جدولها و رکوردها بدرستی ایجاد شده اند.
